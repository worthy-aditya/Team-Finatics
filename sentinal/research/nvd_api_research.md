NVD API Research
1. Purpose

The National Vulnerability Database (NVD) is the U.S. government's official repository of publicly disclosed cybersecurity vulnerabilities. It is maintained by the National Institute of Standards and Technology (NIST) and provides structured vulnerability information based on the Common Vulnerabilities and Exposures (CVE) standard.

The NVD API enables applications to programmatically retrieve vulnerability data, including:

CVE identifiers
Vulnerability descriptions
CVSS severity scores
CWE (Common Weakness Enumeration) information
Affected products and configurations
References to security advisories
Publication and modification dates
Purpose in SentinelAI CLI

SentinelAI CLI will use the NVD API to enrich scan results generated by security tools such as Nmap. After identifying software, services, or known CVEs, SentinelAI will query the NVD API to obtain detailed vulnerability information. This allows the application to present users with meaningful explanations, severity ratings, remediation guidance, and industry-standard references.

Instead of displaying only raw scan results, SentinelAI will provide actionable security insights for cybersecurity students, IT administrators, and small security teams.

2. Base URL

The NVD API Version 2.0 uses the following base endpoint:
https://services.nvd.nist.gov/rest/json/cves/2.0
All CVE-related requests are sent to this endpoint using the HTTP GET method.

Example:
GET https://services.nvd.nist.gov/rest/json/cves/2.0

3. Endpoint

Primary Endpoint
GET /rest/json/cves/2.0
Purpose
Retrieves vulnerability information stored in the National Vulnerability Database.

The endpoint supports searching by:
CVE ID
Product name
Vendor name
Keywords
Publication date
Modification date
Severity
CPE identifiers

This single endpoint provides access to the complete CVE dataset.

4. Query Parameters

The API supports several query parameters that filter search results.

a) cveId
Searches for a specific vulnerability.

Example:
GET /rest/json/cves/2.0?cveId=CVE-2021-41773

Use Case
When SentinelAI detects an exact CVE ID from another security tool, this parameter retrieves detailed information for that vulnerability.

b) keywordSearch
Searches vulnerabilities using keywords.

Example
keywordSearch=Apache

Possible searches

Apache
OpenSSH
Windows
nginx
OpenSSL

Use Case
Useful when Nmap identifies software versions instead of CVE identifiers.

c) resultsPerPage
Controls how many records are returned in one request.

Example
resultsPerPage=20

Default values can be increased depending on the application requirements.

d) startIndex
Supports pagination.

Example
startIndex=0

Next page

startIndex=20

This is useful when retrieving a large number of vulnerabilities.

e) pubStartDate
Returns vulnerabilities published after a specified date.

Example
pubStartDate=2025-01-01T00:00:00.000

f) pubEndDate
Returns vulnerabilities published before a specified date.

Example
pubEndDate=2025-12-31T23:59:59.999

g) lastModStartDate
Returns vulnerabilities modified after the specified date.

Useful for checking recently updated CVEs.

h) lastModEndDate
Returns vulnerabilities modified before the specified date.

5. Authentication
The NVD API can be accessed without authentication.
However, NIST strongly recommends using an API Key.

Benefits include:

Higher request limits
Better performance
Reduced throttling
Reliable production usage
API Key Storage

For SentinelAI CLI, the API key should never be hardcoded.
Instead, store it inside a .env file.

Example
NVD_API_KEY=your_api_key_here

Python example

import os

API_KEY = os.getenv("NVD_API_KEY")

This keeps credentials secure and prevents accidental exposure on GitHub.

6. Rate Limits
The NVD API enforces request limits to prevent abuse.

Typical limits are:

Without API Key

Approximately 5 requests every 30 seconds

With API Key

Approximately 50 requests every 30 seconds
Why Rate Limits Matter

SentinelAI may need to look up multiple vulnerabilities during a single scan.

For example,

Nmap finds 25 services.
Several services have known vulnerabilities.
SentinelAI performs multiple API requests.

Without proper rate limiting, requests may fail or be temporarily blocked.

Recommended implementation:

Cache previous responses.
Avoid duplicate requests.
Retry failed requests with exponential backoff.


7. JSON Response Structure
The API returns responses in JSON format.

Simplified example

{
  "resultsPerPage": 1,
  "startIndex": 0,
  "totalResults": 1,
  "vulnerabilities": [
    {
      "cve": {
        "id": "CVE-2021-41773",
        "published": "2021-10-05",
        "lastModified": "2024-11-21",
        "descriptions": [
          {
            "lang": "en",
            "value": "Path traversal vulnerability."
          }
        ],
        "metrics": {
          "cvssMetricV31": [
            {
              "cvssData": {
                "baseScore": 9.8,
                "baseSeverity": "CRITICAL"
              }
            }
          ]
        },
        "references": [
          {
            "url": "https://nvd.nist.gov/vuln/detail/CVE-2021-41773"
          }
        ]
      }
    }
  ]
}

The response contains both metadata and vulnerability information.

8. Important Fields
id

Unique CVE identifier.

Example

CVE-2021-41773
descriptions

Human-readable explanation of the vulnerability.

Used by SentinelAI to generate beginner-friendly reports.

published

Original publication date.

Useful for identifying newly discovered vulnerabilities.

lastModified

Last update date.

Allows SentinelAI to detect updated vulnerability information.

metrics

Contains CVSS scores.

Includes:

Base Score
Attack Vector
Attack Complexity
Privileges Required
User Interaction
Scope
Confidentiality Impact
Integrity Impact
Availability Impact
baseScore

Numerical risk score.

Example

9.8
baseSeverity

Severity classification.

Possible values

LOW
MEDIUM
HIGH
CRITICAL

SentinelAI will display these values directly to users.

weaknesses

Contains CWE identifiers.

Example

CWE-22

Useful for mapping vulnerabilities to software weaknesses.

references

Official links to

Vendor advisories
Security bulletins
Research papers
NVD detail pages

These references increase report credibility.

configurations

Lists affected products using CPE identifiers.

This allows SentinelAI to determine whether detected software versions are vulnerable.
