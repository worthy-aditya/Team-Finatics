Report Template Design
SentinelAI CLI – Security Report Template Design

Version: 1.0
Prepared By: Suraj Yadav
Module: Report Generation & CVE Integration
Project: SentinelAI CLI – AI Security Copilot

1. Purpose

The purpose of this document is to define the standard report structure that will be used by SentinelAI CLI when generating security assessment reports.
This template provides a common format for DOCX, PDF, and Markdown reports generated after a security scan. It ensures consistency, readability, and professional presentation of security findings.
The report is designed to be understandable by both cybersecurity professionals and beginners.

2. Design Goals

The report should:

Present findings in a structured format
Highlight important vulnerabilities clearly
Explain technical issues in simple language
Include industry-standard mappings
Provide actionable remediation steps
Prioritize vulnerabilities according to severity
Support multiple export formats (DOCX, PDF, Markdown)

4. Report Sections
4.1 Cover Page

The cover page provides basic information about the generated report.

Fields
Project Name
Report Title
Report ID
Date Generated
Generated By
Report Version

4.2 Executive Summary

This section provides a high-level overview of the scan.

Fields
Total Findings
Critical Findings
High Findings
Medium Findings
Low Findings
Informational Findings
Overall Risk Score
Overall Security Status

4.3 Target Information

Displays information about the scanned target.

Fields
Target Name
IP Address
Hostname
Operating System
Scan Duration
Scan Type
Tool Used

4.4 Scan Results

Lists every vulnerability discovered during the scan.

Each finding should include:

Finding ID
Finding Name
Description
Severity
Affected Host
Affected Port
Service Name
Detection Time

4.5 AI Analysis
This section contains AI-generated explanations.

The AI should provide:

Plain English explanation
Why the issue is important
Possible attack scenario
Business impact
Technical impact

4.6 CVE Details
Displays information related to known vulnerabilities.

Fields
CVE ID
CVSS Score
CVSS Version
Published Date
Last Modified Date
Vulnerability Description
Reference URL

4.7 MITRE ATT&CK Mapping
Maps findings to MITRE ATT&CK techniques.

Fields
Technique ID
Technique Name
Tactic
Description

Example:
Technique ID : T1021
Technique Name : Remote Services
Tactic : Lateral Movement

4.8 OWASP Top 10 Mapping
Maps applicable findings to OWASP categories.

Fields
OWASP Category
Category Description
Risk Explanation

Example:
A05
Security Misconfiguration


4.9 Remediation

Provides recommended actions for fixing vulnerabilities.
Each recommendation should include:

Immediate Fix
Long-Term Solution
Best Practices
Reference Links

Example

Update OpenSSH
Disable weak ciphers
Restrict SSH using firewall rules
Enable key-based authentication

4.10 Risk Summary
Provides an overall assessment of the target.

Fields
Overall Risk Rating
Security Score
Number of Critical Issues
Number of High Issues
Priority Recommendation


4.11 Appendix
Contains additional technical information.

Include:
Raw Scan Output
Command Executed
Tool Versions
References
Generated Logs


5. Severity Classification
Every finding should be classified according to its severity level.

Severity	     Color	       Description	                                                                   Priority
Critical	    🔴 Red	     Immediate action required. Vulnerability can lead to complete system compromise.   	P1
High	        🟠 Orange	   Serious vulnerability requiring urgent attention.	                                  P2
Medium	      🟡 Yellow	   Moderate security risk that should be addressed.	                                    P3
Low	          🟢 Green	   Minor issue with limited impact.	                                                    P4
Informational	🔵 Blue	   No immediate security impact. Used for informational findings only.	                P5

6. Risk Score Guidelines
Risk Score may be calculated using CVSS values.

CVSS Score	Severity
9.0 – 10.0	Critical
7.0 – 8.9	High
4.0 – 6.9	Medium
0.1 – 3.9	Low
0.0	Informational

The report generation module should support:

All three formats should contain the same report structure while preserving formatting and severity indicators.

10. Conclusion
This report template establishes a standardized format for presenting security findings within SentinelAI CLI. It is designed to improve readability, simplify vulnerability analysis, and provide actionable recommendations for users ranging from cybersecurity students to professional security analysts.
